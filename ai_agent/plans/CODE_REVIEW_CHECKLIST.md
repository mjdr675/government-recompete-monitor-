# CODE REVIEW CHECKLIST
# Run this checklist before every commit. All items must pass.

---

## 1. Scope Check

- [ ] Every changed line directly implements a stated requirement in the task file
- [ ] No files changed that are not listed in the task's DB/API/Frontend Changes sections
- [ ] No new features added beyond what the Requirements section specifies
- [ ] No existing functions renamed, deleted, or moved unless the task requires it
- [ ] `ai_agent/plans/` files are unchanged

---

## 2. Security

- [ ] No SQL built with string formatting — all queries use `?` or `%s` placeholders
- [ ] No user input used in file paths, shell commands, or eval()
- [ ] No secrets, tokens, or passwords in source code or test fixtures
- [ ] New routes that modify state: CSRF token validated (after Task 210 ships)
- [ ] New routes that read org data: `org_id` filter applied to all DB queries
- [ ] New routes requiring auth: returns 302 to `/login` without a session (not 403)
- [ ] New admin-only routes: role check applied, returns 403 for insufficient role
- [ ] No `except Exception: pass` — all exceptions logged or re-raised

---

## 3. Database

- [ ] New `CREATE TABLE` uses `IF NOT EXISTS`
- [ ] New `CREATE INDEX` uses `IF NOT EXISTS`
- [ ] New columns use `DEFAULT` where a null-safe default exists
- [ ] `init_db()` is idempotent — safe to call on an existing database
- [ ] No `DROP TABLE`, `DELETE FROM`, or `ALTER TABLE ... DROP COLUMN` without explicit task requirement
- [ ] FTS rebuild called after any batch upsert operation
- [ ] All queries against `contracts`, `capture_opportunities`, `proposals`,
      `watchlists`, `saved_searches` include `org_id` filter

---

## 4. Routes

- [ ] New route registered with `@app.route` or added to the correct Blueprint
- [ ] Route listed in `_PUBLIC_PATHS` only if it truly requires no authentication
- [ ] GET-only routes do not modify state
- [ ] POST/PATCH/DELETE routes return appropriate status codes (200, 201, 204, 400, 403, 404)
- [ ] JSON-returning routes set `Content-Type: application/json`
- [ ] Routes accepting file uploads validate file extension and size
- [ ] Redirect URLs validated against allowlist (no open redirect)

---

## 5. Templates

- [ ] User-supplied strings rendered with `{{ var }}` (auto-escaped by Jinja2), not `{{ var | safe }}`
- [ ] `| safe` filter only used on HTML generated by the application itself, never on user input
- [ ] Links use `url_for()`, not hardcoded paths
- [ ] New templates extend `base.html`
- [ ] Forms include CSRF token input (after Task 210 ships)

---

## 6. Tests

- [ ] `pytest -q` exits with code 0 (or 5 if no tests collected in a new file)
- [ ] Test count is ≥ test count from the previous session
- [ ] Every new route has at least one test for: success case, auth required, invalid input
- [ ] Every new DB function has at least one test for: success case, empty result
- [ ] No live DB used in tests — `tmp_path` fixture in use
- [ ] External API calls (SAM.gov, Anthropic, Stripe) mocked in all tests
- [ ] No `time.sleep()` in tests

---

## 7. Code Quality

- [ ] No unused imports
- [ ] No unused variables or functions introduced by this task
- [ ] No `print()` statements in application code (use `logging`)
- [ ] No `TODO` comments left in production code paths
- [ ] No commented-out code blocks
- [ ] Functions longer than 50 lines: split if they serve multiple purposes
- [ ] New utility functions placed in the most relevant existing module (not a new file
      unless a new module is explicitly required by the task)

---

## 8. Backwards Compatibility

- [ ] Existing routes return the same response structure as before
- [ ] Existing DB columns not renamed or removed
- [ ] `init_db()` still works on an existing DB without data loss
- [ ] Existing templates still render without error
- [ ] No changes to the `get_contracts()`, `dashboard_analytics()`, or `opportunity_recommendations()`
      signatures unless explicitly required — these are called from tests

---

## 9. Commit Hygiene

- [ ] `git diff --staged` reviewed before commit — no unintended files staged
- [ ] Commit message follows format: `<type>: <description> (Task <number>)`
- [ ] No `.env`, `contracts.db`, `*.pyc`, or `patches/failures/` files staged
- [ ] `TASK_LOG.md` updated with one new row
- [ ] `HANDOFF.md` updated with one new entry
- [ ] Task file moved from `ai_agent/queue/` to `ai_agent/done/`
- [ ] No push to remote (push is a human action)

---

## 10. Quick Disqualifiers (stop and fix before continuing)

Any of these must be fixed before the task is considered complete:

| Issue | Action |
|---|---|
| `pytest` fails | Fix the failing test or revert the change that broke it |
| SQL without parameterization | Rewrite with placeholders |
| Secret in source code | Remove, rotate the secret, add to `.gitignore` |
| Route accessible without auth (unintentional) | Add to `require_login` check |
| `org_id` missing from query on org-scoped table | Add filter |
| File outside task scope modified | Revert that file |
| Test count decreased | Restore deleted tests or add replacements |
